------------------------------------파일병합기 ver 1.0.1---------------------------------------

*주의*
-밑에서 서술할 기능  설명 부분을 반드시 숙지한 뒤 사용해 주시기 바랍니다.
-jsp와 java 작업에서의 편의를 위해 제작한 프로그램입니다.
-버그로 인한 오류가 있을 수 있으며 파일이 손상될 수도 있습니다.
-윗 내용으로 불안하신 분은 프로젝트 폴더를 미리 백업한 뒤 사용해주시기 바랍니다.
(자체적으로 자동 백업 기능이 있긴 합니다.)
-주의사항을 고지했으므로 파일병합기 사용에 따른 손실에 일절 책임을 지지 않습니다.
-프로그램 제작으로 희생시킨 제 시험기간과 수면시간, 학점에 애도를 부탁드립니다.

----------------------------------------※필독※-------------------------------------------------
프로젝트 파일을 통채로 지정하는 것은 권장되지 않습니다-
이유: 체크박스로 지정하지 않은 확장자의 파일(예: jar, imf 등)을 붙여넣는 현상이 파악되었습니다.
따라서 폴더를 전체지정으로 선택하셨다면 백업 폴더에서 원래 파일을 가져와 붙여넣는 방법으로
해결이 가능하나 가급적이면 jar등이 들어있는 폴더는 지정하지 않는 것을 권장합니다.
만약 확장자가 같으나 파일 이름이 다르면 붙여넣어진 파일을 지우는 것만으로 해결이 가능합니다.

기능 설명
1. 기존 폴더 선택
기존에 작업하던 코드가 있는 폴더를 선택합니다
2. 수정된 폴더 선택
기존 폴더에 덮어씌울 폴더를 선택합니다
3. 복사할 파일 선택
복사하고 싶은 파일의 종류만 선택해 복사할 수 있습니다
(복사하고 싶은 파일이 들어있는 폴더의 경우 자동으로 복사됩니다)
4. DB 정보 선택
데이터베이스를 사용하는 경우 
데이터베이스와 연동하는
Connection conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/*", "*", "*");
구문에서 *에 해당하는 부분을 지정할 수 있습니다.
빈칸으로 두는 경우 수정된 파일의 *이 그대로 들어갑니다.
직접 입력하는 경우 반드시 정확하게 입력해주시기 바랍니다.
기본값 버튼을 누르면 전부 수정된 파일의 *대로 입력 됩니다.
5. 병합이 완료되면 결과 화면이 뜹니다.
6. 만일의 사태를 대비해 기존 폴더의 파일 중 수정된 파일은
파일병합기 폴더의 backup 폴더에 _backup이 붙어 자동으로 저장됩니다.
